# OLA 민원 사진 분류 CNN

CCTV 등에서 촬영된 민원 사진을 **불법주정차 · 보행방해물 · 현수막** 3개 카테고리로 분류하는
이미지 분류 모델입니다. 사전학습 가중치 없이 PyTorch로 CNN을 직접 설계·학습했고,
동일 조건의 전이학습(ResNet18)과 비교한 뒤, 두 모델을 모두 ONNX로 변환해
**브라우저에서 서버 없이 직접 추론**(onnxruntime-web)하는 웹 데모까지 연결했습니다.

> 서울시 생활안전 통합 플랫폼 **OLA**의 "사진으로 민원 신고" 기능에 탑재된 모델입니다.
> (국비교육 조별 프로젝트)

## 결과 요약

| 실험 | 모델 | 검증 정확도 | 핵심 변화 |
|---|---|---:|---|
| 1~3차 | 직접 설계 CNN | 61~65% | 정체 — 원인 분석 대상 |
| 4차 | 직접 설계 CNN | **76.3%** | 카테고리별 크롭 배수 적용 (Grad-CAM 분석 기반) |
| 5차 | 직접 설계 CNN (구조 동일) | **86.3%** | 데이터 1.76배 확대 + 학습 레시피 개선 |
| 비교 | 전이학습 ResNet18 | **93.1%** | 동일 데이터·동일 분리, ImageNet 사전학습 |

모델 구조는 4차 이후 한 번도 바꾸지 않았습니다. **성능을 끌어올린 것은 전부
데이터 전처리와 학습 방법**이었고, 이 과정이 이 프로젝트의 핵심 스토리입니다.

## 무엇이 정확도를 올렸나

**1~3차 → 4차 (+15%p): 크롭 파이프라인 수정.**
현수막 recall이 0.49로 유독 낮았는데, 오분류 샘플 82장 전체에 Grad-CAM을 적용해
모델이 현수막이 아니라 주변 배경을 보고 판단한다는 것을 확인했습니다. 원인은 모든
카테고리에 동일하게 적용된 크롭 배수(3.0배) — 불법주정차는 도로 맥락이 필요하지만
현수막은 객체 자체가 식별 대상이라, 카테고리별 배수(3.0/2.5/1.5)로 재크롭했습니다.

**4차 → 5차 (+10%p): 데이터 확대 + 학습 레시피.**
원본 58만여 개 라벨 중 4,800장만 쓰고 있던 것을 8,453장으로 확대하고
(같은 영상은 train/val에 절대 나눠 담기지 않도록 영상 그룹 단위 분리 유지),
증강 강화(RandomResizedCrop + TrivialAugmentWide + RandomErasing),
Label Smoothing, AdamW + Warmup/Cosine, Early Stopping을 적용했습니다.

**전이학습 비교 (93.1%).**
같은 데이터·같은 분리로 ResNet18을 파인튜닝하면 2 epoch 만에 직접 설계 모델을
넘어섭니다. 소규모 데이터에서 사전학습의 이점을 정량적으로 확인했습니다.

자세한 실험 과정(문자 인코딩 이슈, Grad-CAM 분석, 차수별 confusion matrix)은
[`report/`](report/)의 실험분석보고서에 있습니다.

## 웹 데모: 듀얼 모델 브라우저 추론

두 모델 모두 ONNX(opset 18)로 변환해 onnxruntime-web(WASM)으로 브라우저에서
직접 추론합니다. 사진 신고 모달에서 모델을 전환하면 **같은 사진을 두 모델로 즉시
재분석**해 판단을 비교할 수 있습니다.

- 기본 모델: 전이학습 ResNet18 (93.1%, 입력 224×224, 약 44MB)
- 비교 모델: 직접 설계 CNN (86.3%, 입력 160×160, 약 5MB)
- 선택된 모델만 지연 로드, 불법주정차 분류 시 Tesseract.js로 차량번호 OCR

프론트엔드 추론 로직(onnxruntime-web 기반)은 이 저장소에는 포함하지 않았습니다.
